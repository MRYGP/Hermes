# Automation：每日状态漂移审计

## 自动化名称

```text
daily-status-drift-audit
```

## 目标

每天检查学习仓库或 SK 仓库中关键文档是否存在状态漂移、过期结论或缺少下一步动作。

## 触发频率

第一阶段：

```text
人工触发，每日最多一次。
```

后续可升级为：

```text
每天固定时间只读运行。
```

## 输入范围

第一版建议只检查：

- `PROJECT_STATE.md`
- `README.md`
- `docs/00_项目定位.md`
- `project/Knowledge_Files_Checklist.md`
- `project/Repository_Knowledge_Sync_Protocol.md`

## 执行步骤

1. 读取输入范围。
2. 提取当前阶段、当前定位、下一步、知识库上传规则。
3. 检查是否存在互相冲突的定位。
4. 检查是否存在过期入口或旧名称。
5. 检查 `PROJECT_STATE.md` 是否需要更新。
6. 输出报告。

## 输出报告

```text
运行时间：

检查范围：

发现问题：

高风险项：

建议修改：

需要人工确认：
```

## 权限边界

- 默认只读。
- 不自动提交。
- 不自动删除。
- 不自动重写关键文档。
- 写入必须经过用户确认。

## 升级条件

只有满足以下条件才升级为自动写入：

- 已经手动运行 5 次以上。
- 输出稳定。
- 误报率低。
- 有清晰回滚方案。
- 高风险动作有审批。

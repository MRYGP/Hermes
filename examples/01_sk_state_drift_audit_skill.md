# Skill：SK 状态漂移审计

## 适用场景

当需要检查 SK 仓库中关键文档之间是否存在状态不一致、旧结论残留、缺少下一步动作时使用。

## 不适用场景

- 只修改一个明确文件。
- 用户已经给出唯一事实来源，不需要跨文件对比。
- 没有足够文件路径，无法判断状态。

## 默认输入

按真实 SK 仓库结构调整。第一版可使用：

- `README.md`
- `PROJECT_STATE.md`
- `ops/`
- `docs/`
- `project/Knowledge_Files_Checklist.md`

## 默认权限

只读。不得自动改文件，不得自动删除文件，不得自动提交。

## 执行步骤

1. 读取指定文件。
2. 提取每个文件中的当前阶段、最近完成、待办、已废弃入口。
3. 对比同一项目在不同文件中的状态是否一致。
4. 标记旧入口、旧版本、旧结论。
5. 标记缺少下一步动作的任务。
6. 输出问题清单，不直接修复。

## 输出格式

```text
审计范围：

发现的冲突：

疑似过期入口：

缺少下一步动作：

需要用户拍板：

建议由 Codex 执行的修改工单：
```

## 安全边界

- 只读。
- 不把疑似状态当事实。
- 不重写长文。
- 不自动更新看板。
- 不自动提交。

## 验收标准

- 每个问题必须指向具体文件。
- 每个冲突必须说明冲突双方。
- 不确定项必须标记为“待确认”。
- 输出最后必须给出一个最小下一步。

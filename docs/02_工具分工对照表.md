# 工具分工对照表

## 总体判断

当前不应把 Hermes 当成主力生产工具。更合理的分工是：

```text
Codex / Claude Code 负责当下干活。
Hermes 负责理解 Agent OS 如何组织。
ChatGPT Project 负责学习、复盘和沉淀。
```

## 对照表

| 能力 | Codex | Claude Code | Hermes | ChatGPT Project |
| --- | --- | --- | --- | --- |
| 读仓库 | 强 | 强 | 可用 | 依赖上传文件 |
| 改代码 / 改文档 | 强 | 强 | 可用 | 不直接适合本地执行 |
| 生成 patch | 强 | 强 | 可用 | 适合生成方案 |
| 长期学习 | 中 | 中 | 强 | 强 |
| profile 隔离 | 中 | 中 | 强 | 项目级隔离 |
| skills 沉淀 | 强 | 强 | 强 | 可沉淀为指令 |
| quick commands | 中 | 强 | 强 | 弱 |
| cron / automations | 强 | 中 | 强 | 项目内偏弱 |
| hooks / approvals | 强 | 强 | 强 | 依赖平台能力 |
| MCP / connectors | 强 | 强 | 强 | 取决于可用连接器 |
| 成本控制训练 | 中 | 中 | 强 | 中 |
| 作为教材 | 中 | 中 | 强 | 强 |

## 推荐分工

### Codex

用于：

- 仓库治理
- 文件修改
- patch 生成
- 测试验证
- PR / review / CI 修复
- 本地自动化执行

不要用它做：

- 长篇抽象理论沉淀的唯一来源
- 频繁变化状态的长期记忆库

### Claude Code

用于：

- 复杂代码理解
- 大范围重构
- 工程协作
- 大代码库问答

不要用它做：

- 无边界的长期管家
- 没有验证闭环的自动执行

### Hermes

用于：

- 学 profile 隔离
- 学 memory 边界
- 学 skills / cron / gateway 组合
- 学 agent 如何长期演化
- 学成本与安全边界

不要用它做：

- 当前阶段的主力仓库执行器
- 生产环境自动改仓库的唯一入口

### ChatGPT Project

用于：

- 放学习路线
- 放稳定知识文件
- 做复盘
- 做概念对照
- 生成下一步行动

不要用它做：

- 存放高频变化项目状态
- 替代本地仓库真实执行

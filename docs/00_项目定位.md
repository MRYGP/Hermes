# 项目定位

## 一句话定位

**SK Agent OS 学习台** 是一个面向个人开发者的 Agent 工作流学习仓库。

它的目标不是研究所有 AI 工具，也不是让 Hermes 取代 Codex / Claude Code，而是借 Hermes 的机制，把个人 Agent 系统中最关键的结构学清楚：

- 记忆如何管理
- 技能如何沉淀
- 项目规则如何固化
- 自动化如何调度
- 工具权限如何约束
- 成本如何控制
- 多项目上下文如何隔离

## 为什么不是继续叫 SK Hermes 教练

原来的“SK Hermes 教练”定位是：

> 边学 Hermes，边把 SK 仓库用 Hermes 维护和更新。

这个定位在早期合理，但现在需要调整。Codex 和 Claude Code 已经覆盖了很多仓库执行能力，包括读仓库、改文件、跑命令、审查、skills、hooks、MCP、自动化等。

因此 Hermes 不再适合作为默认主力执行层。它更适合作为一个可观察、可拆解、可迁移的 Agent OS 原型机。

## 新定位

```text
SK Agent OS 学习台
= 以 Hermes 为主教材
+ 以 Codex / Claude Code 为现实对照组
+ 以 SK 仓库为落地场景
+ 以 ChatGPT Project 为学习与复盘中心
```

## 当前不做什么

- 不追求“系统学完 Hermes 全功能”。
- 不把 Hermes 设计成主力仓库执行器。
- 不把 GPTS 做成全自动管家。
- 不把频繁变化的状态放进长期知识库。
- 不做与 SK 无关的大型系统设计。

## 当前只做什么

第一阶段只做 5 件事：

1. 建立 Agent OS 的基本概念框架。
2. 梳理 Codex / Claude Code / Hermes / ChatGPT Project 的分工。
3. 学 Hermes 中最可迁移的机制。
4. 把每个机制映射到 SK 仓库动作。
5. 产出最小可运行模板。

## 判断标准

这个仓库不是为了证明 Hermes 先进，而是为了回答：

> 如果我要长期使用 AI 维护一个个人知识与项目系统，我应该如何设计记忆、技能、规则、自动化、权限和成本边界？

# 记忆、技能、规则的边界

很多 Agent 系统混乱，不是因为工具不够强，而是因为把不同类型的信息放错了地方。

## surface 与知识类型不是一回事

Chat、Work、Project、Codex、Hermes 是执行或交互载体。

memory、skill、规则、状态、自动化是信息和流程类型。

不能因为某个 surface 能保存内容，就把所有内容都放入该 surface。

## memory 记事实

memory 适合长期稳定事实。

可以记：

- 用户长期偏好
- 项目长期目标
- 稳定术语
- 协作原则
- 不容易变化的背景

不应该记：

- 当前任务进度
- 今天的判断
- 临时实验结果
- 待办清单
- 高频变化的状态

判断标准：

> 如果这个信息 30 天后大概率仍然成立，才考虑进入 memory。

## skill 记流程

skill 适合记录可重复执行的流程。

可以写：

- 任务目标
- 适用场景
- 输入要求
- 执行步骤
- 输出格式
- 验收标准

不应该写：

- 一次性任务状态
- 大量背景长文
- 模糊愿景
- 需要实时更新的数据

判断标准：

> 如果这个流程未来会重复执行 3 次以上，就考虑做成 skill。

## AGENTS.md 记项目规则

AGENTS.md 适合记录项目级约束。

可以写：

- 目录结构
- 文件修改规则
- 测试命令
- 验证方式
- 禁止事项
- 代码风格

不应该写：

- 产品长文
- 研究材料
- 当前任务状态
- 每周计划
- 大量教程

判断标准：

> 如果 agent 每次进入仓库都必须知道，就写进 AGENTS.md。

## quick command 记机械动作

quick command 适合高频、短小、固定格式的动作。

可以写：

- 打开关键文件
- 输出固定模板
- 执行固定检查
- 生成固定摘要

不应该写：

- 高风险写入
- 开放式推理
- 多步骤复杂判断

判断标准：

> 如果这个动作不需要重新解释，也不需要复杂判断，就适合 quick command。

## PROJECT_STATE.md 记当前状态

`PROJECT_STATE.md` 适合记录当前阶段、已完成、未决问题、下一步和不要重复做。

它不是长期知识文件，也不能被旧聊天或旧上传文件替代。

## ChatGPT Project 记学习上下文

ChatGPT Project 适合保存学习材料、稳定说明、复盘记录。

可以放：

- 学习路线图
- 概念速查表
- 工具对照表
- 稳定模板
- 复盘总结

不应该放：

- 高频变化状态
- 每日临时看板
- 未确认结论
- 会很快过期的外部信息

判断标准：

> 如果它是学习材料或长期有效规则，可以放；如果它是当前状态，放本地仓库或看板，不放长期知识库。

## 易混淆边界

- Work 产物不等于仓库事实。
- Task 输出不等于当前状态。
- App 返回的数据不等于本地真相。

这些内容经过确认并写回本地仓库后，才成为仓库当前事实。

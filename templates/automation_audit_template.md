# Automation Audit Template

## 自动化名称

例如：

```text
daily-status-drift-audit
```

## 目标

这项自动化要周期性检查什么？

```text
检查 SK 仓库中关键文档是否存在状态漂移、过期结论或缺少下一步动作。
```

## 触发频率

```text
每日一次 / 每周一次 / 发布前触发
```

第一阶段建议：

> 人工触发或低频只读，不要直接常驻写入。

## 输入范围

明确列出文件或目录：

```text
- README.md
- docs/
- ops/
```

## 执行步骤

1. 读取指定范围。
2. 提取状态字段。
3. 检查冲突、过期、不完整。
4. 输出报告。
5. 等待用户确认是否修改。

## 输出报告

```text
运行时间：

检查范围：

发现问题：

高风险项：

建议修改：

需要人工确认：
```

## 权限边界

- 默认只读。
- 不自动提交。
- 不自动删除。
- 不自动重写关键文档。
- 写入必须经过用户确认。

## 升级条件

只有满足以下条件才升级为自动写入：

- 已经手动运行 5 次以上。
- 输出稳定。
- 误报率低。
- 有清晰回滚方案。
- 高风险动作有审批。

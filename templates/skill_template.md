# Skill Template

## Skill 名称

填写一个明确动作名，例如：状态漂移审计。

## 适用场景

什么时候应该调用这个 skill？

```text
当用户需要检查多个项目文档之间是否存在状态不一致、旧结论残留、缺少下一步动作时使用。
```

## 不适用场景

什么时候不应该调用？

```text
当用户只是要改一个明确文件，或者当前没有足够上下文判断状态时，不使用本 skill。
```

## 输入

需要用户或仓库提供什么？

- 关键文档路径
- 当前目标
- 需要检查的范围
- 是否允许写入

## 执行步骤

1. 读取指定文档。
2. 提取当前状态、任务、结论、下一步。
3. 对比是否存在冲突。
4. 标记过期或不确定信息。
5. 输出问题清单。
6. 只在用户明确允许时生成修改建议。

## 输出格式

```text
审计范围：

发现的问题：

疑似过期信息：

需要用户确认：

建议下一步：
```

## 安全边界

- 默认只读。
- 不自动删除文件。
- 不自动重写长文。
- 不把疑似状态当成事实。

## 验收标准

- 输出必须指出具体文件。
- 每个问题必须说明原因。
- 不确定的地方必须标记为待确认。
